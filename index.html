01

Static HTML — Single File Drop

Simplest Upload index.html to any static host

No build, no Node, no server. Users supply their own OpenRouter API key from the Settings panel. The entire app — UI, logic, styles — is one file.

bash
# Test locally $ python3 -m http.server 8000 # Open http://localhost:8000/index.html # Or with Node $ npx serve .

02

Next.js Static Export

Recommended Build → deploy the /out folder

next.config.js already has output: 'export' set. The _headers and _redirects files are pre-configured for Netlify and Cloudflare Pages. Users still supply their own OpenRouter key — no server required.

bash
$ npm install $ npm run build # Static output lands in /out # Deploy the /out directory to your host of choice $ npx wrangler pages deploy out # Cloudflare $ netlify deploy --prod --dir=out # Netlify
☁ Cloudflare Pages ◈ Netlify ▲ Vercel 🐙 GitHub Pages
03

Docker Full Stack

Full Featured Frontend + API server via docker compose

Use this when you want server-side features: shared OpenRouter key, AutoTune, dataset collection, multi-user hosting. Two containers — an nginx frontend and a Node API server — orchestrated by Compose.

Browser :3000 ──► web (nginx) :80 │ /v1/* proxy │ ▼ api (Express) :7860 │ OpenRouter API
bash
# 1 · Create your .env file $ cp .env.example .env # Add: OPENROUTER_API_KEY=sk-or-... # Add: GODMODE_API_KEY=your-secret # 2 · Build and launch $ docker compose up --build -d # 3 · Open http://localhost:3000 # To rebuild after code changes $ docker compose up --build --force-recreate
🐳 Any VPS (DigitalOcean / Hetzner / Linode) ☁ Railway 🚀 Fly.io 🤗 HF Spaces (Docker SDK)
API Only HuggingFace Spaces — Dockerfile

Deploy just the Express API to HF Spaces (Docker SDK). The root Dockerfile exposes port 7860, which HF Spaces expects.

bash
# Build locally first to verify $ docker build -t g0dm0d3-api -f Dockerfile . $ docker run -p 7860:7860 \ -e OPENROUTER_API_KEY=sk-or-... \ -e GODMODE_API_KEY=mysecret \ g0dm0d3-api # Then push to HF Spaces repo as a Docker Space # Health check: GET /v1/health → 200 OK
04

Environment Variables

Variable Required Description
OPENROUTER_API_KEY required Your OpenRouter key — powers all model calls (50+ models). Get one at openrouter.ai/keys
GODMODE_API_KEY required Bearer token callers must send to authenticate against the API server. Set to any secret string.
HF_TOKEN optional HuggingFace write token — only needed if you enable opt-in dataset publishing.
HF_DATASET_REPO optional Target HF dataset repo (e.g. yourname/g0dm0d3-research). Pairs with HF_TOKEN.
PORT optional API server port. Defaults to 7860 (HF Spaces standard). Change if running elsewhere.
WEB_PORT optional Host port for the nginx frontend container. Defaults to 3000 in docker-compose.yml.

⚠ Options 1 and 2 need no server-side env vars — users enter their own OpenRouter key directly in the Settings modal; it stays in their browser localStorage and is never sent to your server.
